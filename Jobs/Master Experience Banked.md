### MediTrack — Healthcare Microservices Platform

**Links:** [GitHub](https://github.com/AadityaDiwan/MediTrack) / [Write-up](https://aadityapdiwan.dev/) **Dates:** 2026 **Context:** Personal — production-grade healthcare backend simulating real EHR concerns **Tags:** #java #springboot #kafka #microservices #security #ci-cd

#### What it is

A microservices backend for a fictional hospital system, covering patient registration, laboratory order management, and insurance policy administration. Designed to demonstrate enterprise Java patterns at the level expected in a backend engineering role at a healthcare or fintech company.

#### What you built & how

- Architected three independent services (patient, lab, insurance) using **hexagonal architecture** (ports & adapters), keeping domain logic fully decoupled from Spring, JPA, and Kafka.
- Built a Next.js 15 (App Router) frontend with a full proxy layer — UI routes all API calls through server-side Next.js API routes, keeping backend services and auth tokens completely shielded from the browser; implemented dedicated views for patient management, lab order tracking, critical result flagging, and insurance policies with real-time data from all three microservices
- Implemented **transactional outbox pattern** in the lab service — lab orders write an event to a DB outbox table within the same transaction, and a scheduled relay publishes to Kafka, eliminating message loss on broker outages.
- Built **role-based access control (RBAC)** end-to-end: roles embedded as JWT claims at login, extracted by a custom `OncePerRequestFilter`, and enforced at the HTTP method + path level via Spring Security with `@EnableMethodSecurity`.
- Shipped full insurance policy CRUD with hexagonal layering, a Kafka consumer for `patient.created` events, optimistic locking via `@Version`, and dual Flyway migration paths (H2 for local dev, PostgreSQL for Docker).
- Hardened security: externalised all secrets to environment variables, locked down all endpoints with role rules, **fixed a PHI log-leak** (BasicBinder: TRACE was printing SSNs and diagnoses to logs), and added `@Valid` + constraint annotations to all request DTOs.
- Configured Kafka consumer reliability: **dead-letter topics (DLT) + exponential backoff** (1s → 2s → 4s, 5 attempts) via `DefaultErrorHandler`.
- Set up **GitHub Actions CI** with three parallel jobs (one per service) running build, tests, and OWASP dependency vulnerability scanning on every push and PR.
- Standardised all services on Spring Boot 3.2.4 / Java 21, migrated JJWT from 0.11.5 → 0.12.6 (full API rewrite), added OpenAPI/Swagger UI to all three services.

#### Stack

Java 21, Spring Boot 3.2.4, NextJS, Spring Security (JWT/stateless), Spring Data JPA, Apache Kafka, PostgreSQL, H2, Flyway, Redis, Docker Compose, GitHub Actions CI, OWASP Dependency Check, Lombok, Springdoc OpenAPI, Micrometer/Prometheus, Zipkin/Jaeger
